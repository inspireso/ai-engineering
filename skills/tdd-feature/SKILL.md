# TDD Feature — 测试驱动功能实现

接受功能规格，按严格 TDD 循环实现功能：先写测试，再实现，最后重构。

---

## 输入要求

用户需提供功能规格，格式不限但应包含：

```
功能：[功能名称]
目标模块：[包路径或模块名]

规格要点：
- [具体需求 1]
- [具体需求 2]
- ...

约束/约定：
- [技术约束，如框架、依赖]
```

---

## Phase 1: RED — 生成失败测试

### Step 1.1: 分析规格，提取测试场景

将功能规格拆解为可测试场景：

| 场景 | 测试名 | 预期行为 |
|------|--------|----------|
| 正常流程 | `should[行为]_when[条件]` | 验证主路径 |
| 边界值 | `shouldHandle_[边界情况]` | null、空、最大值 |
| 异常路径 | `shouldThrow_[异常]_when[条件]` | 验证错误处理 |

输出测试场景清单，AskUserQuestion 确认是否覆盖完整。

### Step 1.2: 创建测试文件

```bash
# 确认测试目录结构
ls src/test/java/<目标包路径>/
```

创建测试类，命名约定：`<功能名>Test.java`

### Step 1.3: 编写测试方法

每个场景写一个测试方法：

```java
@Test
void should[ExpectedBehavior]_when[Condition]() {
    // given
    [准备测试数据和环境]
    
    // when
    [调用目标方法]
    
    // then
    [验证结果或异常]
}
```

测试应：
- 使用有意义命名：`should..._when...`
- 包含 given-when-then 结构
- 验证正面和负面情况

### Step 1.4: 运行测试，确认失败

```bash
./mvnw test -Dtest=<测试类名> -q 2>&1 | tail -30
```

输出测试失败摘要：

```
RED 状态确认：
- 场景 1: ✅ 失败（预期）
- 场景 2: ✅ 失败（预期）
- 场景 3: ✅ 失败（预期）

全部测试按预期失败，进入 GREEN 阶段。
```

若有测试意外通过，检查测试是否正确验证了预期行为。

---

## Phase 2: GREEN — 最小实现

### Step 2.1: 逐场景实现

按优先级实现，每次只让一个测试通过：

1. 正常流程 → 先实现
2. 边界值 → 其次
3. 异常路径 → 最后

### Step 2.2: 最小代码原则

只写让当前失败测试通过的代码：
- 不提前优化
- 不添加未测试的功能
- 简单实现优先

### Step 2.3: 每次变更后运行测试

```bash
./mvnw test -Dtest=<测试类名> -q
```

跟踪进度：

```
GREEN 进度：
- 场景 1: ✅ 通过
- 场景 2: ✅ 通过
- 场景 3: ⏳ 待实现
```

### Step 2.4: 全部通过后确认

当所有测试通过：

```
GREEN 状态达成：
- 测试通过: N/N
- 实现文件: [列出新增/修改的生产代码文件]

进入 REFACTOR 阶段。
```

---

## Phase 3: REFACTOR — 优化与清理

### Step 3.1: 识别重构点

检查代码中的：
- 魔法数字 → 提取为常量或配置
- 重复代码 → 提取方法或类
- 命名不当 → 重命名
- 硬编码依赖 → 注入或配置化
- 缺失文档 → 添加必要注释

### Step 3.2: 逐项重构

每项重构后立即运行测试：

```bash
./mvnw test -Dtest=<测试类名> -q
```

若测试失败，立即回滚并分析原因。

### Step 3.3: 跨文件重构前检查

若重构涉及跨文件修改（如重命名类、移动方法），调用 refactor-analysis skill：

```
请先执行 refactor-analysis，分析此重构的影响范围。
```

或手动使用 Grep 搜索引用点：

```bash
grep -r "旧名称" --include="*.java" -l src/
```

### Step 3.4: 最终验证

```bash
# 运行全量测试确保无回归
./mvnw test -pl <模块名> -q 2>&1 | tail -30
```

```
- 全量测试: 通过 ✅

- 测试文件: [路径]

- 配置变更: [如有]
```

- [ ] RED: 所有测试初始失败

- [ ] 全量测试无回归
